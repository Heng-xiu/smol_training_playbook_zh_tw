## [資料策劃的藝術](https://huggingfacetb-smol-training-playbook.hf.space/#the-art-of-data-curation)

想像一下:你花了數週時間完善你的架構、調整超參數,並建立最穩健的訓練基礎設施。你的模型收斂得很漂亮,然後...它無法編寫連貫的程式碼,在基本數學上掙扎,甚至可能在句子中途切換語言。出了什麼問題?答案通常在於資料。雖然我們痴迷於花哨的架構創新和超參數掃描,但資料策劃通常決定我們的模型是否變得真正有用,還是只是另一個昂貴的實驗。這是在隨機網頁爬取上訓練與在精心策劃的高品質資料集上訓練之間的區別,後者實際上教授我們希望模型學習的技能。

如果模型架構定義了你的模型_如何_學習,那麼資料定義了它_學習什麼_,沒有任何計算量或優化器調整可以補償在錯誤內容上的訓練。此外,正確獲得訓練資料不僅僅是擁有好的資料集。它是關於組裝正確的**混合**:平衡衝突的目標(如強大的英語 vs. 穩健的多語言性)並調整資料比例以與我們的效能目標對齊。這個過程不太關於找到普遍最佳混合,而更關於提出正確的問題並設計具體的計劃來回答它們:

  - 我們希望我們的模型擅長什麼?
  - 哪些資料集最適合每個領域,我們如何混合它們?
  - 對於我們的目標訓練規模,我們是否有足夠的高品質資料?

本節是關於使用原則性方法、消融實驗和一點點煉金術的混合來導航這些問題,將一堆優秀的資料集變成一個優秀的訓練混合。
